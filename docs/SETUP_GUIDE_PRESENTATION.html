Four Copilot Agents

🔧

@setup-project

One-time onboarding. Clones hg branch, installs dependencies, configures .env, sets owner identity. Asks generate-only vs generate-and-run mode.

💻

@test-generator

Reads use-case CSV (any sheet). Filters by "UI To-be-automated = Yes". Generates Java test code, data files, constants, locators, utility methods. Compiles.

▶

@test-runner

Runs tests (single or batch). On failure: classifies error type, opens browser via Playwright MCP, inspects UI, fixes code, recompiles, re-runs. Up to 3 auto-heal attempts.

🔎

@test-debugger

Deep investigation mode. Analyzes ScenarioReport.html, opens SDP pages live, creates test data via API, fixes broken XPaths and logic.

Use-Case CSV — Canonical Format

8 columns. All sheets processed. Only UI To-be-automated = Yes rows are picked.

Column Purpose
UseCase IDMaps to test scenario ID — traced in reports
SeverityCritical → HIGH   Major → MEDIUM   Minor → LOW
ModuleParent module path (Admin, CMDB, Changes, Requests...)
Sub-ModuleEntity subclass routing (determines Java file placement)
Impact AreaThese 3 are cumulated — merged together to form the full scenario context. If coverage overflows ~80 lines, methods split with _1, _2 suffixes.

Self-Healing Test Runner

When a test fails, the runner classifies the failure and applies targeted fixes:

🔍
LOCATOR
➔
Opens SDP page via Playwright MCP, takes snapshot, inspects DOM, finds correct XPath, updates Locators.java
🌐
API
➔
Checks REST call response, validates input wrapper, fixes API path or payload in preProcess / APIUtil
🛠
COMPILE
➔
Reads javac error output, fixes missing imports / wrong signatures / syntax errors, recompiles
💡
LOGIC
➔
Analyzes test flow, identifies incorrect assertion or missing wait, adjusts the test method
Each test gets up to 3 auto-heal attempts. After fixing, the code is recompiled and the test is re-run automatically.

What Gets Generated

For each use case, the agent creates or updates multiple files following framework conventions:

☍

Entity.java

@AutomaterScenario annotation with all 9 fields (id, group, priority, dataIds, tags, description, owner, runType, switchOn)

📝

EntityBase.java

Test method implementation with try/catch/finally, report lifecycle, and utility calls

📎

*_data.json

Test input data with $(placeholder) support for dynamic values

📂

*DataConstants.java

Type-safe TestCaseData constants — no raw string literals

🎯

*Locators.java

XPath / By locators as interface constants

🔨

*ActionsUtil.java

Reusable UI operation methods (navigate, click, fill, verify)
