## Key Technical Learnings

### 1. Problem Module UI Override (IMPORTANT CORRECTION)
- Problem is flat-status but STILL REQUIRES `@Override` for `_UI()` methods
- Reason: blank canvas from `createNewBlankWorkflowInCanvas()` has NO status nodes → transition drag loop fails
- Fix: pre-create minimal workflow via API with real status IDs, GET to count `baseCount`, drag `(limit - baseCount + 1)` delta nodes
- DO NOT assume flat-status = no override needed — check if the module's blank canvas needs status nodes

### 2. Asset Workflow = Special Sub-Module Pattern
- All Asset workflow API calls require `sub_module` field in payload
- Base `Workflow.java` never injects `sub_module` → all Asset boundary tests MUST `@Override`
- `Workflow.java` rev 5322 adds Asset to the module→stage-config and API key mappings

### 3. Checkstyle NeedBraces — Inline Catch Blocks FORBIDDEN
- Zoho uses **Checkstyle** (not PMD); rule is `NeedBraces` covering ALL block types
- `} catch (Exception ignore) {}` MUST be expanded to multi-line
- "Error in codecheck invocation" = Zoho's CHECK SERVER IS BROKEN (not a code violation)
- Fixed 9 inline catch blocks across 5 files in rev 5323

### 4. Task Node Connector Ports
- Task FlowNode with `has_error_path: true` needs `output_Completed` / `output_Overdue` ports
- Generic `input` / `output` causes API rejection
- Fixed in rev 5320 for `createWorkflowWithNTasks()` (affected IR_007, IR_009)

### 5. GitHub Push from hg Repo
- `git init` inside hg directory: `.git/` + `.hg/` coexist with no interference
- GitHub "Repository not found" = repo missing OR auth wrong (same misleading error)
- Linux: use `ssh-keyscan -t ed25519 github.com >> ~/.ssh/known_hosts` + add SSH key to GitHub
- Alternative: PAT in HTTPS URL (remove token from URL after push)
- Workflow: `git init` → feature branch → add src/resources → commit → remote add → push
