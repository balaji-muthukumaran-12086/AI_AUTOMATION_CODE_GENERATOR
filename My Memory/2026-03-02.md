## Locator Fixes Applied (via Playwright DOM Inspection)

### Round 1: Container ID fix (ALL locators)
- **Problem**: All locators used `change_associations_linked_changes` (guessed) — doesn't exist
- **Fix**: Changed to `change_associations_parent_change` (actual DOM ID)
- Also fixed: Attach button (`name="associating-change-button"`), dropdown options (`name="associate_parent_change"` / `name="associate_child_changes"`)

### Round 2: Select2 filter dropdown fix
- **Problem**: `FILTER_OPTION` XPath used `//div[contains(@class,'select2-result')]//div[...]` — wrong
- **Actual DOM**: Select2 renders `<li class="select2-result"><div class="select2-result-label">All Changes</div></li>` in a `<div class="select2-drop select2-drop-active">` appended to `<body>` (outside the dialog)
- **Fix**: Changed to `//div[contains(@class,'select2-result-label') and contains(text(),'...')]`
- **Status**: Fix applied to source, compiled ✅, not yet re-run

### Key DOM Structure Discovered

<!-- Select2 dropdown (appended to body, NOT inside dialog) -->
